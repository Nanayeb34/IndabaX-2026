In‑context memory refers to the LLM’s ability to keep track of past interactions by storing a short history in the prompt. Because models have a fixed token limit, we must compress older turns into a concise summary that captures key facts and decisions. This summary is then prepended to the conversation before each new request. The approach keeps the agent’s context fresh, reduces token usage, and prevents the model from repeating or forgetting earlier steps. In practice, a rolling summary is updated after every turn, often using the same LLM to write the condensed summary.
